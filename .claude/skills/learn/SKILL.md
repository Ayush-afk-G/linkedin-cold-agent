name: learn
description: Explore and understand a codebase, file, or concept
---

When asked to learn about something:

1. If it's a file or directory: read it thoroughly, understand the structure, summarize key patterns
2. If it's a concept in the codebase: grep for relevant usage, trace the flow, explain how it works
3. If it's an external concept: explain it in the context of this project
4. Always output a clear summary with:
   - What it does
   - How it connects to other parts of the codebase
   - Key patterns or gotchas
5. Do NOT make any changes to files — this is read-only exploration
